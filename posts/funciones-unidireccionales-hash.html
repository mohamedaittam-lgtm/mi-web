Las funciones hash unidireccionales son algoritmos matemáticos que toman una entrada y producen una salida de longitud fija, generalmente una cadena de números y letras llamada "hash".

Tipos


MD5 (Message Digest 5): MD5 es un algoritmo ampliamente utilizado en el pasado, pero es ahora considerado inseguro debido a vulnerabilidades conocidas que permiten colisiones. No se recomienda su uso para fines de seguridad.

SHA-1 (Secure Hash Algorithm 1): Similar a MD5, SHA-1 también se considera obsoleto y vulnerable a colisiones. Debe evitarse para aplicaciones de seguridad.

SHA-256, SHA-384, y SHA-512: Estas son variantes más seguras de la familia SHA (Secure Hash Algorithm). Ofrecen longitudes de hash más largas y se utilizan comúnmente en aplicaciones de seguridad, como en la firma digital y la verificación de integridad de archivos.

SHA-3: SHA-3 es la última adición a la familia de algoritmos hash seguros, seleccionada a través de un proceso de competencia público. Ofrece una alternativa sólida a las funciones hash anteriores.

bcrypt y scrypt: Estos son algoritmos hash diseñados específicamente para almacenar contraseñas de manera segura. Son resistentes a ataques de fuerza bruta y a ataques de diccionario al incluir un factor de costo o una función de memoria intensiva para dificultar los ataques.

Programas que podemos utilizar 

openssl: Una herramienta de línea de comandos que admite varios algoritmos hash y otros cifrados. Es ampliamente utilizado en sistemas Unix y Linux.

Hashcat: Una potente herramienta de recuperación de contraseñas que puede utilizar GPU para realizar ataques de fuerza bruta o ataques de diccionario contra hashes.

Python hashlib: El módulo hashlib en Python proporciona una interfaz de programación para calcular hashes utilizando varios algoritmos, como SHA-256.

Uso en seguridad informática

Integridad de datos: Las funciones hash se utilizan para verificar que los datos no se han modificado durante la transmisión o el almacenamiento. Si el hash de los datos recibidos coincide con el hash calculado previamente, se puede confiar en la integridad de los datos.

Contraseñas seguras: Los hashes de contraseñas se almacenan en lugar de las contraseñas reales en las bases de datos. Esto protege las contraseñas en caso de una violación de datos, ya que los atacantes no pueden recuperar fácilmente las contraseñas originales a partir de los hashes.

Firma digital: Se utilizan hashes para verificar la autenticidad de un mensaje o archivo firmado digitalmente. Si el hash del mensaje coincide con el hash firmado, se puede verificar que el mensaje no ha sido alterado y que el remitente es auténtico.

Almacenamiento seguro de datos: Las funciones hash también se utilizan en sistemas de almacenamiento distribuido para garantizar la coherencia y la integridad de los datos almacenados en múltiples ubicaciones.
