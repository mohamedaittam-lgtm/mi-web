Active Directory en Windows Server 2022

Controlador de dominio, GPOs, DHCP integrado y unidades organizativas.

00Introducción

Promoverás un Windows Server 2022 a controlador de dominio (AD DS), con DNS integrado, ámbito DHCP, unidades organizativas (OU) y directivas de grupo (GPO). Se incluyen los pasos por GUI y por PowerShell.

ParámetroValor
Dominioempresa.local
Servidor / DCDC01
IP estática192.168.50.10/24
DNS preferido127.0.0.1

01Preparación del servidor

Antes de promover: IP estática, nombre de equipo y actualizaciones.

PowerShell
# PowerShell (como Administrador)
Rename-Computer -NewName "DC01" -Restart

New-NetIPAddress -InterfaceAlias "Ethernet" -IPAddress 192.168.50.10 `
  -PrefixLength 24 -DefaultGateway 192.168.50.1
Set-DnsClientServerAddress -InterfaceAlias "Ethernet" `
  -ServerAddresses 127.0.0.1
Un DC debe tener IP fija y apuntarse a sí mismo como DNS. Nunca uses DHCP para el propio controlador de dominio.

02Instalar el rol AD DS

Por Administrador del servidor → Agregar roles → «Servicios de dominio de Active Directory», o por PowerShell:

bash
Install-WindowsFeature -Name AD-Domain-Services -IncludeManagementTools

Promover a controlador de dominio (bosque nuevo)

PowerShell
Install-ADDSForest `
  -DomainName "empresa.local" `
  -DomainNetbiosName "EMPRESA" `
  -ForestMode "WinThreshold" `
  -DomainMode "WinThreshold" `
  -InstallDns:$true `
  -SafeModeAdministratorPassword (Read-Host -AsSecureString "DSRM password")
# El servidor se reinicia automáticamente.
La contraseña DSRM (Directory Services Restore Mode) es clave para recuperar AD. Guárdala en lugar seguro.

03Verificar AD y DNS

PowerShell
Get-ADDomain
Get-ADForest
nltest /dsgetdc:empresa.local
# DNS: zonas creadas automáticamente
Get-DnsServerZone
Resolve-DnsName empresa.local

AD DS crea automáticamente la zona DNS directa empresa.local y registros SRV de servicio (LDAP, Kerberos). Verifica que existen.

04Unidades organizativas y usuarios

Estructura las OU según la organización para aplicar GPO de forma granular:

PowerShell
# Crear OUs
New-ADOrganizationalUnit -Name "EMPRESA" -Path "DC=empresa,DC=local"
New-ADOrganizationalUnit -Name "Usuarios"  -Path "OU=EMPRESA,DC=empresa,DC=local"
New-ADOrganizationalUnit -Name "Equipos"   -Path "OU=EMPRESA,DC=empresa,DC=local"
New-ADOrganizationalUnit -Name "Servidores" -Path "OU=EMPRESA,DC=empresa,DC=local"

# Crear un usuario
New-ADUser -Name "Ada Lovelace" -SamAccountName "alovelace" `
  -UserPrincipalName "alovelace@empresa.local" `
  -Path "OU=Usuarios,OU=EMPRESA,DC=empresa,DC=local" `
  -AccountPassword (Read-Host -AsSecureString "Pass") `
  -Enabled $true -ChangePasswordAtLogon $true

# Grupo y miembro
New-ADGroup -Name "GG_Contabilidad" -GroupScope Global `
  -Path "OU=Usuarios,OU=EMPRESA,DC=empresa,DC=local"
Add-ADGroupMember -Identity "GG_Contabilidad" -Members "alovelace"

05Directivas de grupo (GPO)

Las GPO aplican configuración a usuarios/equipos de una OU. Ejemplo: política de contraseñas y fondo corporativo.

PowerShell
# Crear y vincular una GPO
New-GPO -Name "GPO-Seguridad-Base"
New-GPLink -Name "GPO-Seguridad-Base" `
  -Target "OU=EMPRESA,DC=empresa,DC=local"

# Forzar actualización en cliente
gpupdate /force
gpresult /r        # ver GPO aplicadas

Configuraciones típicas (editor GPMC)

  • Longitud mínima de contraseña y bloqueo de cuenta.
  • Mapeo de unidades de red por grupo.
  • Restricción de panel de control para usuarios estándar.
  • Despliegue de software (.msi) por OU.
  • Fondo de escritorio y protector de pantalla corporativos.
Diseña GPO pequeñas y específicas; es más fácil de depurar que una GPO gigante. Usa nombres descriptivos.

06DHCP integrado

Instala el rol DHCP en el mismo servidor (o en otro miembro del dominio) y autorízalo en AD:

PowerShell
Install-WindowsFeature DHCP -IncludeManagementTools

# Autorizar en AD (obligatorio en un dominio)
Add-DhcpServerInDC -DnsName "DC01.empresa.local" -IPAddress 192.168.50.10

# Crear ámbito
Add-DhcpServerv4Scope -Name "LAN-Empresa" `
  -StartRange 192.168.50.100 -EndRange 192.168.50.200 `
  -SubnetMask 255.255.255.0 -State Active

# Opciones (gateway y DNS)
Set-DhcpServerv4OptionValue -ScopeId 192.168.50.0 `
  -Router 192.168.50.1 -DnsServer 192.168.50.10 -DnsDomain empresa.local
Seguridad. La autorización en AD evita servidores DHCP no autorizados (rogue) en la red.

07Unir un cliente y verificar

En un Windows 10/11 cliente, configura el DNS apuntando al DC y une al dominio:

cliente
# En el cliente (PowerShell admin)
Add-Computer -DomainName "empresa.local" -Restart
  • El DC responde a Get-ADDomain y la zona DNS existe.
  • OUs, usuarios y grupos creados correctamente.
  • GPO vinculada y aplicada (gpresult /r).
  • Ámbito DHCP activo y autorizado, clientes reciben IP.
  • Cliente unido al dominio e inicia sesión con cuenta de AD.
