00Introducción

Montarás un stack de observabilidad con Prometheus (recolección de métricas), node_exporter (métricas de host), Grafana (dashboards) y Alertmanager (alertas), todo orquestado con Docker Compose.

ComponentePuertoFunción
Prometheus9090Scraping y almacenamiento TSDB
node_exporter9100Métricas de CPU/RAM/disco/red
Grafana3000Visualización y dashboards
Alertmanager9093Enrutado y envío de alertas

01Stack con Docker Compose

docker-compose.yml
# docker-compose.yml
services:
  prometheus:
    image: prom/prometheus:latest
    volumes:
      - ./prometheus.yml:/etc/prometheus/prometheus.yml
      - ./alerts.yml:/etc/prometheus/alerts.yml
      - prom_data:/prometheus
    ports: ["9090:9090"]
    restart: unless-stopped

  node-exporter:
    image: prom/node-exporter:latest
    ports: ["9100:9100"]
    restart: unless-stopped

  alertmanager:
    image: prom/alertmanager:latest
    volumes:
      - ./alertmanager.yml:/etc/alertmanager/alertmanager.yml
    ports: ["9093:9093"]
    restart: unless-stopped

  grafana:
    image: grafana/grafana:latest
    environment:
      - GF_SECURITY_ADMIN_PASSWORD=admin123
    volumes:
      - grafana_data:/var/lib/grafana
    ports: ["3000:3000"]
    restart: unless-stopped

volumes:
  prom_data:
  grafana_data:

02Configurar Prometheus

prometheus.yml
# prometheus.yml
global:
  scrape_interval: 15s
  evaluation_interval: 15s

alerting:
  alertmanagers:
    - static_configs:
        - targets: ["alertmanager:9093"]

rule_files:
  - "alerts.yml"

scrape_configs:
  - job_name: "prometheus"
    static_configs:
      - targets: ["localhost:9090"]

  - job_name: "node"
    static_configs:
      - targets: ["node-exporter:9100"]
bash
docker compose up -d
docker compose ps
# Targets en estado UP:
# http://localhost:9090/targets

03Reglas de alerta

Define alertas con PromQL. Ejemplos: host caído, CPU alta y poco disco.

alerts.yml
# alerts.yml
groups:
  - name: infra
    rules:
      - alert: HostDown
        expr: up == 0
        for: 1m
        labels: { severity: critical }
        annotations:
          summary: "Host {{ $labels.instance }} caído"

      - alert: CPUAlta
        expr: 100 - (avg by(instance)(rate(node_cpu_seconds_total{mode="idle"}[5m])) * 100) > 85
        for: 5m
        labels: { severity: warning }
        annotations:
          summary: "CPU > 85% en {{ $labels.instance }}"

      - alert: DiscoBajo
        expr: (node_filesystem_avail_bytes{mountpoint="/"} / node_filesystem_size_bytes{mountpoint="/"}) * 100 < 15
        for: 10m
        labels: { severity: warning }
        annotations:
          summary: "Menos del 15% de disco en {{ $labels.instance }}"
bash
# Recargar reglas sin reiniciar:
docker compose kill -s SIGHUP prometheus

04Alertmanager (notificaciones)

Enruta las alertas a un canal (email, Slack, Telegram...). Ejemplo con email:

alertmanager.yml
# alertmanager.yml
route:
  receiver: "equipo"
  group_by: ["alertname", "instance"]
  group_wait: 30s
  repeat_interval: 3h

receivers:
  - name: "equipo"
    email_configs:
      - to: "ops@empresa.local"
        from: "alertas@empresa.local"
        smarthost: "smtp.empresa.local:587"
        auth_username: "alertas@empresa.local"
        auth_password: "secreto"
        require_tls: true
Para Slack usa slack_configs con un webhook entrante; para Telegram, telegram_configs con bot token y chat_id.

05Grafana: datasource y dashboards

  1. Accede a http://localhost:3000 (admin / admin123).
  2. Connections → Data sources → Add Prometheus → URL http://prometheus:9090 → Save & test.
  3. Dashboards → New → Import → ID 1860 (Node Exporter Full) → selecciona el datasource.

El dashboard 1860 muestra CPU, memoria, disco, red y carga del sistema listos para usar. Crea paneles propios con consultas PromQL:

PromQL
# Uso de memoria %
100 * (1 - node_memory_MemAvailable_bytes / node_memory_MemTotal_bytes)
# Tráfico de red (recibido)
rate(node_network_receive_bytes_total[5m])
# Carga a 1 minuto
node_load1

07Verificación

  • Todos los targets en UP en /targets.
  • Datasource Prometheus conectado en Grafana.
  • Dashboard Node Exporter mostrando datos en vivo.
  • Regla de alerta visible en /alerts (Prometheus).
  • Alertmanager recibe y enruta las alertas.
