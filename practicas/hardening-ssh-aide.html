Hardening SSH + auditoría con AIDE

Claves, 2FA, integridad de ficheros y reglas de auditoría avanzadas.

⏱ 2h ⚡ Dificultad: Alta LinuxSSHAIDE

Contenido

00Introducción

Endurecerás el acceso SSH al máximo (claves, 2FA con TOTP, restricciones) y desplegarás AIDE (Advanced Intrusion Detection Environment) para vigilar la integridad de ficheros críticos del sistema y generar reglas de auditoría avanzadas.

Esta práctica complementa el hardening básico de Ubuntu Server. Hazla sobre un servidor ya actualizado.

01Autenticación solo por clave

Genera una clave robusta en el cliente y despliégala:

cliente
ssh-keygen -t ed25519 -a 100 -C "admin@bastion"
ssh-copy-id admin@servidor

Endurece /etc/ssh/sshd_config:

sshd_config
Protocol 2
PermitRootLogin no
PasswordAuthentication no
PubkeyAuthentication yes
AuthenticationMethods publickey
PermitEmptyPasswords no
MaxAuthTries 3
MaxSessions 2
LoginGraceTime 20
ClientAliveInterval 300
ClientAliveCountMax 2
AllowUsers admin
X11Forwarding no
AllowAgentForwarding no
# Solo algoritmos modernos:
KexAlgorithms curve25519-sha256,curve25519-sha256@libssh.org
Ciphers chacha20-poly1305@openssh.com,aes256-gcm@openssh.com
MACs hmac-sha2-512-etm@openssh.com,hmac-sha2-256-etm@openssh.com
bash
sudo sshd -t && sudo systemctl restart ssh
Verifica siempre con sshd -t y mantén una sesión abierta hasta confirmar el acceso.

02Segundo factor (TOTP)

Añade 2FA con Google Authenticator (TOTP) además de la clave SSH:

bash
sudo apt install libpam-google-authenticator -y
# Como el usuario que accede:
google-authenticator
# Responde: y (time-based), escanea el QR con la app,
# guarda los códigos de recuperación.

Configura PAM y SSH para exigir clave y TOTP:

/etc/pam.d/sshd
# /etc/pam.d/sshd  (añadir al inicio)
auth required pam_google_authenticator.so
sshd_config
# /etc/ssh/sshd_config
KbdInteractiveAuthentication yes
AuthenticationMethods publickey,keyboard-interactive
bash
sudo systemctl restart ssh
Doble factor real. Con AuthenticationMethods publickey,keyboard-interactive exiges ambos factores: clave + código TOTP.

03Capas extra de protección

  • fail2ban con jail sshd (ver práctica de Ubuntu hardening).
  • UFW limitando el puerto SSH: sudo ufw limit OpenSSH.
  • Cambiar el puerto por defecto reduce ruido de bots (no es seguridad real, pero ayuda).
  • Restringir por IP de origen con Match Address en sshd_config o con UFW.
bash
# Acceso SSH solo desde la red de gestión:
sudo ufw allow from 192.168.50.0/24 to any port 22 proto tcp

04Instalar e inicializar AIDE

AIDE crea una base de datos de huellas (hashes, permisos, tamaños) de los ficheros y detecta cualquier modificación.

bash
sudo apt install aide aide-common -y
# Inicializar la base de datos de referencia:
sudo aideinit
# Activar la base recién creada:
sudo mv /var/lib/aide/aide.db.new /var/lib/aide/aide.db
Línea base. Genera la base AIDE justo después de instalar y endurecer el sistema, cuando sabes que está «limpio». Esa será tu línea base de confianza.

05Reglas de auditoría

Las reglas en /etc/aide/aide.conf definen qué atributos vigilar por ruta:

/etc/aide/aide.conf
# Grupos de comprobación (atributos)
# p=permisos i=inode n=nlinks u=user g=group
# s=size m=mtime c=ctime + hashes sha256/sha512
NORMAL = p+i+n+u+g+s+m+c+sha256
LOG    = p+i+n+u+g

# Rutas críticas a vigilar
/etc        NORMAL
/bin        NORMAL
/sbin       NORMAL
/usr/bin    NORMAL
/boot       NORMAL
/root       NORMAL

# Directorios que cambian: solo metadatos o ignorar
/var/log    LOG
!/var/log/journal
!/proc
!/sys
!/dev
bash
# Validar configuración y regenerar base tras cambios legítimos:
sudo aide --config-check
sudo aide --init
sudo mv /var/lib/aide/aide.db.new /var/lib/aide/aide.db

06Comprobaciones y automatización

Ejecuta una comprobación manual y compárala con la base de referencia:

bash
sudo aide --check
# Ejemplo de salida cuando algo cambió:
#   File: /etc/passwd
#   Mtime: old = ...  new = ...
#   SHA256: old = ...  new = ...

Comprobación diaria automática + informe

El paquete aide-common instala un cron diario. Para un informe propio por correo:

bash
sudo nano /etc/cron.d/aide-check
/etc/cron.d/aide-check
30 4 * * * root /usr/bin/aide --check | mail -s "AIDE $(hostname) $(date +\%F)" admin@empresa.local
Tras actualizaciones del sistema, muchos ficheros cambiarán legítimamente: actualiza la base con aide --update para evitar falsos positivos.

07Auditoría del kernel con auditd

Complementa AIDE con auditd para registrar accesos a ficheros en tiempo real:

bash
sudo apt install auditd audispd-plugins -y
# Vigilar cambios en ficheros sensibles:
sudo auditctl -w /etc/passwd  -p wa -k cambios-usuarios
sudo auditctl -w /etc/shadow  -p wa -k cambios-pass
sudo auditctl -w /etc/ssh/sshd_config -p wa -k cambios-ssh
# Persistir reglas:
echo '-w /etc/ssh/sshd_config -p wa -k cambios-ssh' | sudo tee -a /etc/audit/rules.d/asir.rules
sudo systemctl restart auditd
# Consultar eventos:
sudo ausearch -k cambios-ssh --start today

Checklist de seguridad

  • SSH solo por clave Ed25519, root deshabilitado.
  • 2FA TOTP exigido junto a la clave.
  • Algoritmos criptográficos modernos forzados.
  • Base AIDE creada sobre sistema limpio.
  • Comprobación AIDE diaria con informe.
  • auditd registrando accesos a ficheros críticos.
