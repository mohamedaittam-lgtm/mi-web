Instalación y hardening de Ubuntu Server 22.04

Guía completa: instalación mínima, SSH, UFW, fail2ban y actualizaciones automáticas.

00Introducción y objetivos

En esta práctica realizarás una instalación mínima de Ubuntu Server 22.04 LTS y aplicarás una base de hardening (endurecimiento) sólida: usuario sin privilegios con sudo, acceso SSH por clave, cortafuegos UFW, protección contra fuerza bruta con fail2ban y actualizaciones de seguridad automáticas.

Requisitos. Necesitas: una máquina física o virtual con 1 vCPU / 1 GB RAM mínimo, la ISO de Ubuntu Server 22.04 LTS y conexión a Internet.

Resultado esperado

  • Servidor accesible solo por clave SSH (sin contraseña ni root).
  • Solo los puertos imprescindibles abiertos.
  • Intentos de intrusión bloqueados automáticamente.
  • Parches de seguridad aplicados sin intervención.

01Instalación mínima

Arranca desde la ISO y sigue el instalador Subiquity. Las decisiones clave:

  • Idioma y teclado según tu región.
  • Tipo de instalación: Ubuntu Server (minimized) no es obligatorio, pero no marques snaps adicionales.
  • Red: configura IP estática si será un servidor (recomendado).
  • Particionado: usa LVM para flexibilidad futura.
  • Perfil: crea tu usuario (ej. asir) — nunca uses root.
  • Marca «Install OpenSSH server» para administrar en remoto.
  • No instales paquetes server snaps que no vayas a usar.
Buena práctica. La IP estática evita que el servidor cambie de dirección tras un reinicio del DHCP, algo crítico para SSH y servicios.

Configurar IP estática con Netplan (si no lo hiciste en el instalador)

bash
sudo nano /etc/netplan/00-installer-config.yaml
/etc/netplan/00-installer-config.yaml
network:
  version: 2
  ethernets:
    eth0:
      addresses:
        - 192.168.1.20/24
      routes:
        - to: default
          via: 192.168.1.1
      nameservers:
        addresses: [1.1.1.1, 9.9.9.9]
bash
sudo netplan apply
ip a            # comprueba la IP
ip route        # comprueba la puerta de enlace

02Primeras actualizaciones y usuario

Tras el primer arranque, actualiza el sistema completo y verifica tu usuario sudo:

bash
sudo apt update && sudo apt full-upgrade -y
sudo apt autoremove --purge -y
# Comprueba que tu usuario tiene sudo:
groups            # debe aparecer 'sudo'
hostnamectl       # info del sistema
Si necesitas crear un usuario administrador adicional:
bash
sudo adduser operador
sudo usermod -aG sudo operador

03Acceso SSH por clave

La autenticación por clave es mucho más segura que la contraseña. Genera el par de claves en tu equipo cliente, no en el servidor:

cliente
# En tu PC (Linux/macOS/WSL):
ssh-keygen -t ed25519 -C "asir@portatil"
# Copia la clave pública al servidor:
ssh-copy-id asir@192.168.1.20

Prueba la conexión sin contraseña: ssh asir@192.168.1.20. Una vez que funcione, endurece el servicio SSH:

bash
sudo nano /etc/ssh/sshd_config
/etc/ssh/sshd_config
PermitRootLogin no
PasswordAuthentication no
PubkeyAuthentication yes
ChallengeResponseAuthentication no
X11Forwarding no
MaxAuthTries 3
LoginGraceTime 30
AllowUsers asir operador
bash
sudo systemctl restart ssh
# Verifica la sintaxis antes de cerrar la sesión actual:
sudo sshd -t && echo "config OK"
Cuidado. Deja una sesión abierta mientras pruebas en otra terminal. Si te bloqueas, podrás revertir. Nunca desactives PasswordAuthentication sin antes confirmar que tu clave funciona.

04Cortafuegos con UFW

UFW (Uncomplicated Firewall) es la interfaz sencilla de iptables. Política por defecto restrictiva:

bash
sudo ufw default deny incoming
sudo ufw default allow outgoing
sudo ufw allow OpenSSH          # o: sudo ufw allow 22/tcp
# Si tendrás web:
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable
sudo ufw status verbose
Importante. Asegúrate de permitir SSH antes de hacer ufw enable o perderás el acceso remoto.

Limitar SSH (rate limiting)

UFW puede limitar conexiones repetidas, una capa extra contra fuerza bruta:

bash
sudo ufw limit OpenSSH

05Protección con fail2ban

fail2ban analiza los logs y banea IPs que fallan repetidamente la autenticación.

bash
sudo apt install fail2ban -y
sudo cp /etc/fail2ban/jail.conf /etc/fail2ban/jail.local
sudo nano /etc/fail2ban/jail.local
/etc/fail2ban/jail.local
[DEFAULT]
bantime  = 1h
findtime = 10m
maxretry = 4
backend  = systemd

[sshd]
enabled = true
port    = ssh
maxretry = 3
bantime  = 2h
bash
sudo systemctl enable --now fail2ban
sudo fail2ban-client status
sudo fail2ban-client status sshd      # IPs baneadas
Para desbanear una IP: sudo fail2ban-client set sshd unbanip 1.2.3.4

06Actualizaciones automáticas

unattended-upgrades aplica los parches de seguridad sin intervención manual.

bash
sudo apt install unattended-upgrades -y
sudo dpkg-reconfigure -plow unattended-upgrades
sudo nano /etc/apt/apt.conf.d/50unattended-upgrades
50unattended-upgrades
Unattended-Upgrade::Allowed-Origins {
    "${distro_id}:${distro_codename}-security";
    "${distro_id}ESMApps:${distro_codename}-apps-security";
};
Unattended-Upgrade::Automatic-Reboot "true";
Unattended-Upgrade::Automatic-Reboot-Time "04:00";
Unattended-Upgrade::Remove-Unused-Dependencies "true";
bash
# Prueba en seco:
sudo unattended-upgrade --dry-run --debug

07Verificación y checklist

Comprueba que todo el endurecimiento está activo:

bash
sudo ufw status verbose
sudo systemctl status fail2ban ssh
sudo sshd -T | grep -Ei 'permitrootlogin|passwordauth'
systemctl status unattended-upgrades

Checklist de hardening

  • Root login deshabilitado y sin contraseña SSH.
  • Acceso únicamente por clave Ed25519.
  • UFW activo con política deny por defecto.
  • fail2ban vigilando el jail sshd.
  • Parches de seguridad automáticos + reinicio programado.
  • Sistema completamente actualizado.
