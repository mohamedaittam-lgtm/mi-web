Virtualización con Proxmox VE

Instalación, VMs, contenedores, redes VLAN y políticas de backup.

⏱ 3h ⚡ Dificultad: Media ProxmoxKVMLXC

Contenido

00Introducción

Proxmox VE es una plataforma de virtualización de tipo 1 basada en Debian que combina KVM (máquinas virtuales) y LXC (contenedores) con una interfaz web unificada. Instalarás el hipervisor, crearás VMs y contenedores, configurarás redes VLAN y definirás una política de backups.

Requisitos. Requisitos: CPU x86-64 con virtualización (VT-x/AMD-V) activada en BIOS, mínimo 8 GB RAM y 32 GB de disco.

01Instalación del hipervisor

  1. Descarga la ISO de Proxmox VE desde proxmox.com y crea un USB de arranque.
  2. Arranca e instala: selecciona disco (recomendado ZFS si hay varios discos), zona horaria y red.
  3. Define IP estática, gateway y un FQDN (ej. pve.lab.local).
  4. Tras reiniciar, accede a la consola web en https://IP:8006.

Post-instalación: repos sin suscripción

Si no tienes licencia, configura el repositorio no-subscription:

bash
# Deshabilitar repo enterprise:
sudo sed -i 's/^deb/#deb/' /etc/apt/sources.list.d/pve-enterprise.list
# Añadir no-subscription:
echo "deb http://download.proxmox.com/debian/pve bookworm pve-no-subscription" \
  | sudo tee /etc/apt/sources.list.d/pve-no-sub.list
sudo apt update && sudo apt full-upgrade -y

02Redes y VLAN

Proxmox usa Linux bridges (vmbr0) como switch virtual. Para segmentar tráfico, activa VLAN aware en el bridge.

bash
sudo nano /etc/network/interfaces
/etc/network/interfaces
auto vmbr0
iface vmbr0 inet static
    address 192.168.30.2/24
    gateway 192.168.30.1
    bridge-ports eno1
    bridge-stp off
    bridge-fd 0
    bridge-vlan-aware yes
    bridge-vids 2-4094
bash
sudo ifreload -a
802.1Q. Con VLAN aware activo, en cada VM/CT indicas el VLAN Tag (ej. 30) en la pestaña de red. El switch físico debe tener el puerto en modo trunk.

03Subir ISOs y plantillas

Sube imágenes ISO y plantillas de contenedor desde la interfaz o por CLI:

bash
# Plantillas LXC disponibles:
pveam update
pveam available | grep ubuntu
pveam download local ubuntu-22.04-standard_22.04-1_amd64.tar.zst

Las ISO se suben en Datacenter → almacenamiento «local» → ISO Images.

04Crear una máquina virtual (KVM)

Desde la web: Create VM. Parámetros recomendados:

ParámetroValor recomendado
OSISO subida (ej. Ubuntu Server 22.04)
SystemBIOS: OVMF (UEFI) o SeaBIOS; Machine: q35
DiscoSCSI + VirtIO SCSI single, 32 GB, discard activado
CPU2 cores, tipo «host» para mejor rendimiento
RAM2048 MB (ballooning activado)
Redvmbr0, modelo VirtIO, VLAN Tag si aplica

Crear la misma VM por CLI

bash
qm create 100 --name web01 --memory 2048 --cores 2 \
  --net0 virtio,bridge=vmbr0,tag=30 --scsihw virtio-scsi-single
qm set 100 --scsi0 local-lvm:32
qm set 100 --ide2 local:iso/ubuntu-22.04.iso,media=cdrom
qm set 100 --boot order=ide2;scsi0
qm start 100
Instala los qemu-guest-agent dentro de la VM para backups consistentes y apagado limpio.

05Crear un contenedor (LXC)

Los contenedores LXC son más ligeros que las VMs, ideales para servicios Linux:

bash
pct create 200 local:vztmpl/ubuntu-22.04-standard_22.04-1_amd64.tar.zst \
  --hostname ct-web --memory 1024 --cores 1 \
  --rootfs local-lvm:8 \
  --net0 name=eth0,bridge=vmbr0,tag=30,ip=192.168.30.50/24,gw=192.168.30.1 \
  --unprivileged 1 --features nesting=1
pct start 200
pct enter 200      # entrar al contenedor
Usa contenedores unprivileged siempre que sea posible: mayor aislamiento y seguridad.

06Política de backups

Proxmox integra vzdump. Configura un job programado en Datacenter → Backup → Add:

  • Storage: un almacenamiento dedicado (NFS, PBS o disco aparte).
  • Schedule: diario a las 02:00.
  • Selection: todas las VMs/CT o por pool.
  • Mode: snapshot (sin parar la VM, requiere guest agent).
  • Retention: keep-daily=7, keep-weekly=4, keep-monthly=3.
  • Compression: ZSTD.

Backup manual por CLI

bash
vzdump 100 --mode snapshot --compress zstd --storage local
# Restaurar:
qmrestore /var/lib/vz/dump/vzdump-qemu-100-*.vma.zst 100
Para entornos serios usa Proxmox Backup Server (PBS): deduplicación, cifrado e incrementales reales.

07Checklist final

  • Hipervisor accesible en https://IP:8006 y actualizado.
  • Bridge VLAN-aware funcionando con tags.
  • Al menos una VM KVM y un contenedor LXC operativos.
  • Guest agent instalado en las VMs.
  • Job de backup programado con retención.
  • Restauración probada al menos una vez.
