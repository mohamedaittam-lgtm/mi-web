Docker Compose para entornos de desarrollo

Orquestación local de servicios web, BBDD y volúmenes persistentes.

00Introducción

Con Docker Compose levantarás un entorno de desarrollo completo y reproducible en un solo comando: un servicio web (Nginx) como reverse proxy, una app, una base de datos y volúmenes persistentes, todo conectado en una red interna.

Compose define infraestructura local en un único YAML. Ideal para que todo el equipo trabaje con el mismo entorno.

01Instalar Docker y Compose

bash
# Docker Engine + plugin Compose (Ubuntu)
curl -fsSL https://get.docker.com | sudo sh
sudo usermod -aG docker $USER   # re-login para aplicar
docker version
docker compose version
Desde Docker moderno se usa docker compose (plugin), no el antiguo docker-compose.

02Estructura del proyecto

árbol
dev-env/
├── docker-compose.yml
├── .env
├── nginx/
│   └── default.conf
├── app/
│   ├── Dockerfile
│   └── index.php
└── db/
    └── init.sql
.env
# .env
COMPOSE_PROJECT_NAME=devenv
MYSQL_ROOT_PASSWORD=rootpass
MYSQL_DATABASE=appdb
MYSQL_USER=appuser
MYSQL_PASSWORD=apppass
El fichero .env permite parametrizar sin tocar el YAML. Añádelo a .gitignore si contiene secretos.

03El docker-compose.yml

docker-compose.yml
services:
  web:
    image: nginx:1.27-alpine
    ports:
      - "8080:80"
    volumes:
      - ./nginx/default.conf:/etc/nginx/conf.d/default.conf:ro
    depends_on:
      - app
    networks: [frontend]

  app:
    build: ./app
    environment:
      DB_HOST: db
      DB_NAME: ${MYSQL_DATABASE}
      DB_USER: ${MYSQL_USER}
      DB_PASS: ${MYSQL_PASSWORD}
    depends_on:
      db:
        condition: service_healthy
    networks: [frontend, backend]

  db:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: ${MYSQL_ROOT_PASSWORD}
      MYSQL_DATABASE: ${MYSQL_DATABASE}
      MYSQL_USER: ${MYSQL_USER}
      MYSQL_PASSWORD: ${MYSQL_PASSWORD}
    volumes:
      - db_data:/var/lib/mysql
      - ./db/init.sql:/docker-entrypoint-initdb.d/init.sql:ro
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]
      interval: 10s
      timeout: 5s
      retries: 5
    networks: [backend]

volumes:
  db_data:

networks:
  frontend:
  backend:

04Configurar los servicios

Reverse proxy de Nginx hacia la app:

nginx/default.conf
# nginx/default.conf
server {
    listen 80;
    server_name localhost;
    location / {
        proxy_pass http://app:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
app/Dockerfile
# app/Dockerfile
FROM php:8.3-cli
WORKDIR /var/www
COPY index.php .
EXPOSE 8000
CMD ["php", "-S", "0.0.0.0:8000"]
app/index.php
// app/index.php
<?php
$link = @mysqli_connect(getenv('DB_HOST'), getenv('DB_USER'),
                        getenv('DB_PASS'), getenv('DB_NAME'));
echo $link ? "App OK · Conexión a BBDD correcta"
           : "App OK · BBDD no disponible: ".mysqli_connect_error();
db/init.sql
-- db/init.sql
CREATE TABLE IF NOT EXISTS visitas (
  id INT AUTO_INCREMENT PRIMARY KEY,
  ts TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

05Levantar y operar

bash
# Construir y arrancar en segundo plano:
docker compose up -d --build
# Estado y logs:
docker compose ps
docker compose logs -f app
# Probar:
curl http://localhost:8080
# Entrar a un contenedor:
docker compose exec db mysql -uappuser -papppass appdb
# Parar (mantiene volúmenes):
docker compose down
# Parar y BORRAR volúmenes (datos):
docker compose down -v
Persistencia. down conserva el volumen db_data: tus datos persisten entre reinicios. Solo down -v los elimina.

06Buenas prácticas y verificación

  • Usa healthchecks y depends_on: condition para arranques ordenados.
  • Separa redes frontend/backend: la BBDD no se expone al exterior.
  • Fija versiones de imagen (mysql:8.0, no latest).
  • Monta el código como volumen para hot-reload en desarrollo.
  • No publiques el puerto de la BBDD salvo que sea necesario.

Checklist final

  • docker compose up levanta web + app + db.
  • La web responde en http://localhost:8080.
  • La app conecta correctamente con MySQL.
  • Los datos persisten tras down + up.
  • La BBDD solo es accesible desde la red interna.
